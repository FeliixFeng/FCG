# FCG 项目状态（PROJECT_STATUS）

> 更新时间：2026-04-21

## 项目概况

- 项目：Family Care Guardian（家庭健康管理系统）
- 架构：Spring Boot 3 + Vue 3 + MySQL
- 目标：围绕家庭成员“用药计划 -> 今日任务 -> 打卡记录 -> 健康数据”形成闭环

## 当前已完成（以现代码为准）

### 核心业务

- 药品管理：药箱 CRUD、OCR 识别、计划创建
- 首页任务中心：
  - 下一条待处理主任务卡
  - 待处理任务列表
  - 今日已处理列表
  - 剂量与单位展示（如“每次 2 片”）
- 计划联表逻辑：
  - 首页按计划生成当天任务
  - 叠加记录状态（无记录时也可直接打卡）
- 打卡与库存：
  - 首次打卡/跳过可直接创建记录
  - 状态变为“已服”时自动扣库存（按计划剂量，最少 1）
- 健康模块：体征录入、趋势图、周报

### 权限与成员

- 家庭级账号 + 成员切换
- 管理员可查看/切换其他家庭成员数据
- 受控成员支持关怀模式

### 数据一致性

- `med_record` 已有唯一索引：
  - `(user_id, plan_id, scheduled_date, slot_name)`
- 避免同用户同计划同日期同时间段重复记录

## 当前业务链路（关键）

1. 药品页创建 `med_plan`
2. 首页调用 `/api/medicine/plan/records?scheduledDate=...`
3. 后端按 `med_plan` 生成今日任务，叠加 `med_record` 状态
4. 前端打卡：
   - 有 `recordId`：更新记录
   - 无 `recordId`：创建记录
5. 若状态更新为“已服”，后端执行库存扣减
